HTML · CSS · Tailwind

A friendly intro to how the three work together to build web pages.

① HTML — Structure

HTML is like the skeleton of a webpage. It defines what elements exist: headings, paragraphs, buttons, images, links. It says what things are, not what they look like.

🏗️

Elements

Tags like <h1>, <p>, and <div> create the building blocks of your page.

🔗

Attributes

Extra info on elements. href on a link, src on an image, class for styling.

📐

Semantics

Use meaningful tags: <header>, <main>, <section>. It helps accessibility and SEO.

② CSS — Appearance

CSS is the paint and furniture. It makes things look good — colors, spacing, fonts, layout. Every visual property you see on the web is CSS.

/* CSS Example */

.card {

background: white;

border-radius: 12px;

padding: 20px;

box-shadow: 0 2px 8px rgba(0,0,0,0.1);

}

The Box Model — most important CSS concept

Every HTML element is a rectangular box. From outside in:

margin — space outside border — the edge padding — space inside content — the actual stuff

③ Tailwind CSS — Utility Classes

Tailwind is CSS, but pre-written as tiny single-purpose classes. Instead of writing color: indigo in a CSS file, you just write text-indigo-600 directly on the element. Faster, more consistent.
